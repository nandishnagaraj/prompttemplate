# DES-02 — Database Schema Design

**Purpose:** Produce ER model + SQL DDL + normalization rationale + migration notes.

```text
Design a relational database schema for:

Domain: [DOMAIN DESCRIPTION]
Key Entities derived from requirements:
[LIST MAIN ENTITIES]

Generate:
1. Entity-Relationship description (list entities, attributes, relationships)
2. SQL CREATE TABLE statements with:
   - Appropriate data types
   - Primary and foreign keys
   - Indexes for anticipated query patterns
   - NOT NULL / UNIQUE constraints
   - Created_at / updated_at audit columns
3. Explain normalization decisions (1NF, 2NF, 3NF)
4. Identify any denormalization tradeoffs for performance
5. Migration strategy notes

Tech stack: [DB TECHNOLOGY AND VERSION]
```
