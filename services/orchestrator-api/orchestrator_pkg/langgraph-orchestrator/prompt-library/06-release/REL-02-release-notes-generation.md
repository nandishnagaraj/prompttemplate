# REL-02 — Release Notes Generation

**Purpose:** Generate release notes for multiple audiences (users, devs, ops).

```text
Generate user-friendly release notes for version [VERSION]:

Completed user stories:
[PASTE US TITLES AND DESCRIPTIONS]

Bug fixes:
[LIST BUG FIXES]

Target audiences:
- End users (non-technical)
- Developers / API consumers
- Operations team

Generate three versions:
1. USER CHANGELOG — Plain English, benefit-focused, no jargon
2. DEVELOPER CHANGELOG — Technical details, API changes, migration steps
3. INTERNAL RELEASE SUMMARY — Covers all changes, risks, rollback info

Format each with sections:
New Features | Improvements | Bug Fixes | Breaking Changes | Known Issues
```
