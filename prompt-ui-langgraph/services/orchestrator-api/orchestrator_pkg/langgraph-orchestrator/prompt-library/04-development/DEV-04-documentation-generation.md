# DEV-04 — Documentation Generation

**Purpose:** Produce docstrings, inline comments, and README module docs with examples.

```text
Generate comprehensive documentation for:

<code>
[PASTE CODE FILE OR FUNCTION]
</code>

Generate:
1. Function/class-level docstrings (JSDoc / Python docstring format)
2. Inline comments for complex logic blocks
3. README section describing this module:
   - Purpose and responsibilities
   - Public API (inputs, outputs, side effects)
   - Usage examples (at least 2)
   - Error conditions and how they're surfaced
   - Dependencies

Follow [DOCSTRING FORMAT: JSDoc / Google Style / NumPy / etc.]
```
