# US-02 — Acceptance Criteria Deep Dive

**Purpose:** Expand a story into concrete, testable acceptance criteria across functional + NFR dimensions.

```text
For this user story, generate comprehensive acceptance criteria:

Story: [PASTE USER STORY]

Generate:
1. Happy Path Scenarios (3-5 Given/When/Then)
2. Error Scenarios (invalid input, network failure, unauthorized)
3. Edge Cases (boundary values, empty states, concurrent users)
4. Performance Criteria (response time, load)
5. Security Criteria (authentication, authorization, data validation)
6. Accessibility Criteria (WCAG compliance points)

Be specific and testable. Avoid vague language like "should work correctly".
```
