# PRD-03 — PRD Gap Analysis & Validation

**Purpose:** Stress-test a PRD for ambiguity, gaps, conflicts, and untestable requirements.

```text
Review the following PRD critically:

<prd>
[PASTE PRD HERE]
</prd>

Perform a thorough analysis and identify:
1. AMBIGUITIES — Requirements that are unclear or could be interpreted multiple ways
2. GAPS — Missing requirements based on the stated goals
3. CONFLICTS — Requirements that contradict each other
4. MISSING NON-FUNCTIONALS — Security, performance, accessibility gaps
5. UNTESTABLE REQUIREMENTS — Requirements that cannot be verified

For each issue, provide:
- Location (section/requirement number)
- Issue description
- Suggested resolution

Format as a structured review table.
```
