# PRD Quality Checklist

- [ ] Problem statement is clear and backed by data
- [ ] User personas defined with goals and pain points
- [ ] Every functional requirement has measurable acceptance criteria
- [ ] NFRs include performance benchmarks
- [ ] Out-of-scope items explicitly listed
- [ ] Dependencies/blockers documented
- [ ] Success metrics are quantifiable (not “improve performance”)
- [ ] Risks have owners and mitigations
- [ ] Tech constraints reviewed by engineering lead
